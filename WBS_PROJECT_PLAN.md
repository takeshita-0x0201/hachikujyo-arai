# 蜂駆除あらい リニューアルプロジェクト WBS

**プロジェクト名**: 蜂駆除あらい Webサイトリニューアル  
**目的**: 蜂シーズン（5〜10月）に月40〜50件の問い合わせを獲得し、赤字脱却  
**作成日**: 2026-04-02  
**制約**: 2026年5月のシーズン開始まで約1ヶ月  
**追加要件**: 現在の他業者管理サーバーから自社契約のCoreserver CORE-Yへ移行

### デプロイ方法の選択肢

| 方法 | メリット | デメリット |
|------|---------|-----------|
| **All-in-One WP Migration** | 簡単。.wpressで丸ごと移行 | 毎回全体をエクスポート/インポート |
| **FTP差分アップロード** | 変更ファイルだけ転送 | 手動管理が必要 |
| **Git + デプロイスクリプト** | バージョン管理・差分管理 | 初期設定が必要 |
| **WP Migrate DB + SFTP** | DB+ファイルを個別に管理 | プラグイン依存 |

→ 初回は **All-in-One WP Migration** で丸ごと移行、以後の差分は **FTP差分アップロード** が現実的。

### ステージング環境の注意事項

- **noindex必須**: ステージングがGoogleにインデックスされると重複コンテンツになる
- **Basic認証推奨**: 第三者にステージングを見られないようにする
- **本番と同じパーマリンク構造にする**: クライアント確認の精度を上げるため
- **画像パスの注意**: 本番ドメインの絶対パスで画像を参照している場合、ステージングで表示されない可能性あり → Search & Replace DBプラグインでURL置換

## リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| クライアントからの情報取得が遅延 | Phase 0の遅延→全体遅延 | 4/3までに全情報を一括依頼 |
| **ドメイン管理権限が他業者にある** | 本番移行が進まない | 早期にレジストラ移管 or DNS変更権限を取得 |
| **旧業者が解約に応じない/違約金** | 移行が遅延/コスト増 | 契約内容を事前確認。並行稼働も検討 |
| **DNS浸透に時間がかかる** | 最大72時間のダウンタイムリスク | 旧サーバーを浸透完了まで維持 |
| **メール移行漏れ** | 問い合わせメールが届かない | 移行前にメール設定を完了・テスト |
| **クライアント確認のレスポンスが遅い** | 修正ループが長引き全体遅延 | 確認期限を事前合意（各ループ最大2日） |
| **ステージングと本番で表示が異なる** | 本番公開後に想定外の表示崩れ | 同じPHP/WPバージョンに揃える。hosts確認を挟む |
| **無料サーバーの制限** | 容量やPHP制限でWPが動かない | 97MBなので基本OK。ダメなら別の無料サーバーに切替 |
| AIOSEOのv4アップデートで設定消失 | SEO設定のやり直し | v3のまま進め、シーズン後にv4移行 |
| SEO効果が出るまで時間がかかる | 5月の問い合わせ目標未達 | ポータルサイト+リスティング広告で補完 |
| WP Fastest Cacheで変更が反映されない | 修正後に旧ページが表示 | 各Phase完了時にキャッシュクリア |
